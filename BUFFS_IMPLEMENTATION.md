# Реализация системы баффов для оруженосца (SquireAbility)

## Обзор

Реализована полная система баффов для тяжёлых воинов (Heavy Infantry) с использованием паттерна **Декоратор**. Лёгкие пехотинцы (Light Infantry) получили специальную способность оруженосца, которая применяет баффы на соседних тяжёлых воинов.

## Новые компоненты

### 1. Интерфейсы для баффов

#### IBuff (Abilities/IBuff.cs)
Интерфейс для описания любого баффа. Каждый бафф имеет:
- `Name` - название баффа
- `AttackModifier` - модификация атаки
- `DefenseModifier` - модификация защиты
- `ArrowDefenseModifier` - модификация защиты от стрел

#### IBuffable (Abilities/IBuffable.cs)
Интерфейс для юнитов, поддерживающих баффы. Методы:
- `AddBuff(IBuff buff)` - добавить бафф
- `GetActiveBuffs()` - получить список активных баффов
- `RemoveRandomBuff()` - удалить случайный бафф
- `HasBuffs()` - проверить наличие баффов

### 2. Классы баффов (Decorator Pattern)

Все баффы реализуют интерфейс `IBuff`:

- **ShieldBuff** (Abilities/Buffs/ShieldBuff.cs) - добавляет защиту от удара
- **SpearBuff** (Abilities/Buffs/SpearBuff.cs) - добавляет силу атаки
- **HorseBuff** (Abilities/Buffs/HorseBuff.cs) - добавляет и атаку и защиту
- **HelmetBuff** (Abilities/Buffs/HelmetBuff.cs) - добавляет защиту от стрел (уменьшает урон на 2)

Каждый бафф хранит значение бонуса доступа к боевыми характеристикам.

### 3. BuffableHeavyInfantry (Models/Units/BuffableHeavyInfantry.cs)

Новый класс тяжёлого воина, реализующий `IBuffable`:

**Ключевые особенности:**
- Хранит базовые значения атаки и защиты
- Переопределяет свойства `Attack` и `Defense` для включения модификаторов баффов
- Метод `CalculateDamageWithArrowDefense()` для расчёта урона с учётом защиты от стрел
- Хранит список активных баффов
- Поддерживает неограниченное количество баффов

**Пример расчёта характеристик:**
```
Базовая атака: 37
Бафф Копьё (+3) + Бафф Конь (+2) = +5
Итоговая атака: 37 + 5 = 42
```

### 4. SquireAbility (Abilities/SquireAbility.cs)

Специальная способность оруженосца (Light Infantry):

**Механика:**
- Срабатывает с низкой вероятностью (по умолчанию 25%)
- Ищет соседних тяжёлых воинов (спереди и сзади)
- Если соседей несколько, выбирает того, у кого больше HP
- При активации применяет случайный бафф с бонусом от 1 до 5

**Методы:**
- `SetUnitPosition()` - установить позицию оруженосца в армии
- `GetAvailableTargets()` - получить список соседних тяжёлых воинов
- `TryActivate()` - проверить вероятность срабатывания
- `Activate()` - применить бафф к целевому юниту

## Интеграция с боевой системой

### Battle.cs

Добавлены следующие изменения:

1. **ExecuteTurn() обновлён** - добавлена проверка потери баффа при ударе:
   ```csharp
   if (defender is IBuffable buffableDefender && buffableDefender.HasBuffs())
   {
       if (_random.Next(100) < 20) // 20% вероятность
       {
           buffableDefender.RemoveRandomBuff();
       }
   }
   ```

2. **ExecuteSquireAbility()** - новый метод для выполнения способностей оруженосцев:
   - Проверяет вероятность срабатывания
   - Выбирает цель с максимальным HP при наличии нескольких
   - Применяет бафф и выводит информацию

3. **Вызов в ExecuteTurn():**
   ```csharp
   ExecuteArcherAttack(currentArmy, opponentArmy);
   ExecuteWizardClone(currentArmy);
   ExecuteHealerAttack(currentArmy);
   ExecuteSquireAbility(currentArmy);  // Новый вызов
   ```

### ArrowAbility.cs

Обновлена для учёта защиты от стрел:
```csharp
if (target is BuffableHeavyInfantry buffableTarget)
{
    actualDamage = buffableTarget.CalculateDamageWithArrowDefense(actualDamage);
}
```

### BattleDisplayer.cs / IBattleDisplay.cs

Добавлены методы для визуализации способности оруженосца:
- `ShowSquireNoTargets()` - оруженосец не нашел цели
- `ShowSquireBuffAttempt()` - попытка применить бафф
- `ShowSquireBuffSuccess()` - успешное применение баффа
- `ShowSquireBuffFailed()` - неудачная попытка
- `ShowBuffLost()` - информация о потере баффа
- Обновлён `GetAllUnitsAbilities()` для отображения информации о SquireAbility

## Изменения в существующих компонентах

### LightInfantry (Models/Units/LightInfantry.cs)
- Теперь создаётся с SquireAbility

### UnitFactories.cs
- HeavyInfantryFactory теперь создаёт BuffableHeavyInfantry

### ArmyBuilder.cs
- При восстановлении армии тяжёлые воины создаются как BuffableHeavyInfantry

### GameManager.cs
- При загрузке сохранённой игры тяжёлые воины создаются как BuffableHeavyInfantry

### CloneAbility.cs
- Поддерживает клонирование BuffableHeavyInfantry
- При клонировании копирует все активные баффы

## Использованные паттерны

### 1. Decorator Pattern (Паттерн Декоратор)
- **Компонент:** BuffableHeavyInfantry с базовыми характеристиками
- **Декораторы:** ShieldBuff, SpearBuff, HorseBuff, HelmetBuff
- Каждый бафф добавляет модификаторы к базовым характеристикам
- Баффы могут накладываться друг на друга (стакируются)

### 2. Strategy Pattern
- SquireAbility стратегически выбирает цель и тип баффа

## Примеры использования

### Пример 1: Применение баффа оруженосцем
```
- Лёгкий воин (оруженосец) находится повинности рядом с тяжёлым
- Вероятность срабатывания 25%
- Если сработало:
  - Выбирается случайный бафф (например, Щит)
  - Генерируется бонус от 1 до 5 (например, 3)
  - Тяжёлому воину добавляется +3 защиты
```

### Пример 2: Накопление баффов
```
Изначально:
- Атака: 37, Защита: 12

После 1-го баффа Щит (+2):
- Атака: 37, Защита: 14

После 2-го баффа Копьё (+3):
- Атака: 40, Защита: 14

После 3-го баффа Конь (+2):
- Атака: 42, Защита: 16
```

### Пример 3: Потеря баффа при ударе
```
- Тяжёлый воин с 2 баффами получает урон
- Вероятность потери баффа 20%
- Если произошла потеря:
  - Случайно выбирается один из 2 баффов
  - Этот бафф удаляется
```

## Характеристики системы

| Аспект | Значение |
|--------|---------|
| Вероятность срабатывания SquireAbility | 25% (настраивается) |
| Диапазон бонуса баффа | 1-5 |
| Вероятность потери баффа при ударе | 20% |
| Максимум баффов на юнита | Неограничено |
| Типов баффов | 4 |

## Заключение

Реализована полнофункциональная система баффов с использованием паттерна Декоратор, интегрированная в существующую боевую систему. Оруженосцы теперь могут активно влиять на боевой процесс, усиливая своих товарищей. Баффы добавляют стратегическую глубину: слабый лёгкий воин может превратить хорошего тяжёлого воина в убийцу, если сработают его способности.
